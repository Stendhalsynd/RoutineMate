# S4-1 Spec - 리마인더/미기록 감지

## 범위
- 설정 페이지에서 리마인더 정책(활성, 고정 시간, 미기록 감지 시간, 채널, 타임존)을 관리한다.
- API: `GET/POST /v1/reminders/settings`, `GET /v1/reminders/evaluate`.
- 미기록 기준: `meal=0 && workout=0 && bodyMetric=0`.

## Given / When / Then
1. Given 세션이 유효하면, When 리마인더 설정을 저장하면, Then 동일 설정이 재조회된다.
2. Given 당일 기록이 없으면, When evaluate를 호출하면, Then `isMissingLogCandidate=true`를 반환한다.
3. Given 당일 기록을 추가하면, When evaluate를 다시 호출하면, Then `isMissingLogCandidate=false`로 변경된다.

## 실패 테스트
- 잘못된 시간 형식(`25:99`) 저장 시 400.
- 세션 미존재 시 404.

## 완료 기준
- 설정 저장/재조회/평가 API가 통합테스트 green.
