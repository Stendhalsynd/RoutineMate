# S6-1 Spec - 모바일 bootstrap 우선 초기 로딩 + 세션 확인 경량화

## 범위
- 모바일 앱 시작 시 `/v1/bootstrap`를 주 초기 로딩 인터페이스로 사용한다.
- 마지막 성공 세션과 bootstrap 스냅샷을 로컬에 저장하고, 앱 시작 직후 provisional UI를 먼저 복원한다.
- `GET /v1/auth/session`은 빠른 세션 확인 역할만 수행하고, 조회 시점의 Google canonical repair는 수행하지 않는다.
- 모바일 초기 진입에서는 중복 hydration과 range prefetch를 피하고, 필요한 데이터만 1회 로드한다.

## Given / When / Then
1. Given 이전 실행에서 bootstrap 스냅샷이 저장되어 있을 때,
   When 앱을 다시 실행하면,
   Then 네트워크 응답 전에도 마지막 세션/대시보드/당일 기록이 provisional 상태로 먼저 보인다.
2. Given 모바일이 초기 진입할 때,
   When bootstrap을 1회 요청하면,
   Then 대시보드/goal/day/template/reminder 데이터가 함께 적재된다.
3. Given Google 세션 사용자가 `/v1/auth/session`을 조회할 때,
   When 세션 확인만 필요한 경우,
   Then canonical repair를 다시 수행하지 않고 세션을 바로 반환한다.

## 실패 테스트
- stale bootstrap cache가 유효한 provisional 스냅샷으로 복원되면 실패.
- `/v1/bootstrap?view=records`가 `goal`을 비워 두면 실패.
- `/v1/auth/session` 조회 시 `resolveCanonicalGoogleSession()`이 호출되면 실패.

## 완료 기준
- 모바일 초기 진입 시 체감 로딩이 개선되고, 이전 세션이 즉시 보이며 백그라운드 검증으로 최신화된다.
- bootstrap 기반 초기 적재 테스트와 경량 auth/session 회귀 테스트가 green이다.
