# S4-8 - 템플릿 단순화 + 선택형 입력 + fresh 동기화

## Given
- 식단 템플릿은 슬롯별 분리가 아닌 전역 템플릿으로 사용한다.
- 운동 체크인도 식단과 동일하게 템플릿 기반 1탭 기록을 우선한다.
- 대시보드는 기록 직후 stale 캐시 대신 최신 값을 우선 반영해야 한다.

## When
1. 사용자가 식단/운동 체크인에서 `함`을 선택한다.
2. 사용자가 기록 후 즉시 대시보드로 이동한다.
3. 사용자가 모바일에서 날짜/시간/수치를 입력한다.
4. Notion MealTemplates DB에서 `MealSlot` 컬럼이 제거된 상태로 앱을 실행한다.

## Then
1. 활성 템플릿이 0개면 식단/운동 `함` 버튼은 비활성화된다.
2. `completed=true` 저장은 유효한 활성 템플릿 `templateId` 없이는 400을 반환한다.
3. 식단 템플릿 1개 이상이면 아침/점심/저녁/저녁2 모두 동일 템플릿 목록이 표시된다.
4. 기록 mutation 이후 대시보드 조회는 `fresh=1`로 최신 값이 반영된다.
5. 모바일 입력은 텍스트 직접 입력이 아닌 선택형 바텀시트(날짜/시간/수치)로 동작한다.
6. 웹 날짜 필드는 필드 전체 클릭으로 선택 가능해야 한다.
7. MealTemplates에서 `MealSlot` 누락 시에도 템플릿 CRUD/조회가 정상 동작한다.

## 실패 테스트 우선 항목
1. `POST /v1/meal-checkins` with `completed=true` + no templateId -> 400
2. `POST /v1/meal-checkins` with inactive templateId -> 400
3. `POST /v1/workout-checkins` with `completed=true` + no templateId -> 400
4. MealTemplates schema check where `MealSlot` missing -> pass

## UX 수용 기준
1. 모바일 대시보드에 미니 트렌드바가 노출된다.
2. 모바일 기록/설정의 날짜/시간/수치 필드는 picker로만 변경된다.
3. 웹 기록/설정의 날짜 필드는 입력창 전체 클릭 시 캘린더 선택이 열린다.
