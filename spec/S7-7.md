# S7-7 DESIGN.md And Neo Brutalism Tokens

## Goal
- Establish a documented design system for RoutineMate so future web/mobile redesign work follows one Neo Brutalism visual contract.

## Scope
- Add a root `DESIGN.md`.
- Expand shared UI tokens for color, border, and shadow semantics.
- Align web global variables and core shared tokens to the documented design direction.

## Acceptance Criteria
1. `DESIGN.md` defines the visual language, color system, shape rules, and implementation mapping.
2. `packages/ui` exposes shared token values that match the documented design contract.
3. Web global CSS reflects the same tokenized palette and brutalist framing style.

## Verification
- `npm run typecheck`
- `npm run build --workspace @routinemate/web`
