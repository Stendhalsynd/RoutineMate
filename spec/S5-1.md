# S5-1 Spec - 대시보드 로컬 날짜 경계(UTC 밀림) 보정

## 범위
- 대시보드 집계 기준일을 서버 UTC `now`가 아닌 요청 `date(YYYY-MM-DD)` 우선으로 처리한다.
- `GET /v1/dashboard`, `GET /v1/bootstrap` 모두 `date` 쿼리를 받아 동일 기준일로 집계한다.
- 집계 엔진은 `endDateKey`를 직접 받아 윈도우를 계산한다.

## Given / When / Then
1. Given KST 오전(UTC 전일) 시각, When `date=2026-03-04`로 대시보드 조회, Then 2026-03-04 오전 운동이 즉시 집계된다.
2. Given 기록 직후 대시보드 재조회, When `fresh=1&date=today` 호출, Then 같은 날짜 기준으로 합계/버킷이 일관되게 반환된다.
3. Given 모바일/웹이 같은 `date`를 전송, When 동일 세션으로 조회, Then 집계 결과가 동일하다.

## 실패 테스트
- `date`가 잘못된 형식이면 400.
- 세션 미존재면 404(`/v1/dashboard`) 또는 `session: null`(`/v1/bootstrap`).

## 완료 기준
- UTC 경계 재현 케이스(예: KST 08:40)에서 당일 오전 운동 카운트가 누락되지 않는다.
- `apps/web/tests/dashboard-aggregation.test.ts`, `apps/web/tests/api-routes.test.ts` 관련 케이스가 green.
