# S6-2 Spec - stale response 차단 + mutation 재조회 안정화

## 범위
- `loadDayData`와 대시보드 동기화는 최신 요청만 state를 갱신할 수 있어야 한다.
- mutation 직후의 전체 재조회는 즉시 여러 번 겹치지 않도록 debounce 한다.
- optimistic state는 오래된 GET 응답에 의해 되돌려지지 않아야 한다.

## Given / When / Then
1. Given 같은 날짜에 대한 요청이 연속으로 발생할 때,
   When 먼저 보낸 요청이 나중에 도착하면,
   Then 오래된 응답은 `day` state를 덮지 못한다.
2. Given 대시보드 재동기화가 겹칠 때,
   When 이전 요청보다 최신 요청이 먼저/나중에 도착하더라도,
   Then 최신 요청만 반영된다.
3. Given 식단/운동/체성분 mutation 직후,
   When 여러 refresh가 짧은 시간 안에 연속 예약되면,
   Then 실제 전체 재조회는 debounce 윈도우 기준으로 1회만 실행된다.

## 실패 테스트
- 최신 sequence가 아닌 응답이 적용되면 실패.
- debounce delay가 음수이거나 즉시 중복 재실행되면 실패.

## 완료 기준
- 기록 탭에서 `함/안함` 토글이 깜빡이지 않는다.
- body metric 저장 후 항목이 사라졌다 다시 나타나는 현상이 재현되지 않는다.
- sequence/debounce 유틸 테스트가 green이다.
