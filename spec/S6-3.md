# S6-3 Spec - 체중/체지방 입력 UX 개선

## 범위
- 모바일 기록/설정 화면의 체중/체지방 입력은 `최근값`, `-0.1`, `+0.1`, `정밀 선택` 흐름을 제공한다.
- 모바일 정밀 선택은 `십의자리 | 일의자리 | 소수 첫째자리` 3열 입력을 사용한다.
- 모바일 digit 선택은 0~9가 반복되는 형태로 해석되고, 범위를 벗어나는 조합은 막는다.
- 웹은 긴 decimal `<select>` 대신 숫자 입력 + stepper 버튼(`-0.1`, `+0.1`)을 사용한다.

## Given / When / Then
1. Given 현재값이 비어 있고 최근 기록이 있을 때,
   When 모바일 사용자가 `최근값`을 누르면,
   Then 마지막 기록값이 즉시 입력된다.
2. Given 현재값 또는 최근값이 있을 때,
   When 모바일 사용자가 `-0.1` 또는 `+0.1`을 누르면,
   Then 범위 내에서 0.1 단위로 조정된다.
3. Given 모바일 사용자가 digit sheet에서 숫자를 선택할 때,
   When 조합이 허용 범위를 벗어나면,
   Then 해당 조합은 비활성 처리되거나 clamp되어 저장되지 않는다.
4. Given 웹 사용자가 체중/체지방 입력 필드를 사용할 때,
   When stepper 버튼을 누르거나 숫자를 직접 입력하면,
   Then 0.1 단위 숫자 입력으로 동일 값이 반영된다.

## 실패 테스트
- digit wrap이 0~9 범위를 순환하지 않으면 실패.
- quick action이 최근값 fallback 또는 0.1 조정을 올바르게 적용하지 않으면 실패.
- 범위를 넘는 digit/value가 clamp되지 않으면 실패.
- 웹 stepper가 0.1 단위 조정을 보장하지 못하면 실패.

## 완료 기준
- 모바일 체중/체지방 입력에서 긴 정수 범위를 위아래로 탐색하지 않고 빠르게 최근값/미세 조정을 할 수 있다.
- 웹 기록/목표 화면에서 긴 select 없이 숫자 입력과 stepper로 값을 수정할 수 있다.
- 관련 유틸 테스트와 앱 타입체크가 green이다.
