# S5-3 Mobile Header & Navigation - 상태/버튼 정리

## 배경
대시보드 상단에서 불필요한 상태/동기화 텍스트와 모바일 탭이 화면 점유율을 과점해 핵심 콘텐츠가 축소됨.

## 범위
- 상단 고정 영역에서 동기화 상태 배지 제거.
- Google 로그인 버튼만 상단 액션으로 유지.
- Google 연결 상태 텍스트는 상단 브랜드 영역(로고 아래)에서만 표시.
- 모바일은 상단 네비게이션 탭/메뉴 제거 후 사이드바로 이동.

## Given / When / Then
1. Given 사용자 진입 시,
   When 웹 대시보드를 렌더링하면,
   Then “Google 재인증” 혹은 “Google 로그인” 버튼만 보이고 동기화 배지/상태 텍스트는 상단 버튼 그룹에서 보이지 않아야 한다.

2. Given 동일 사용자 계정 표시 텍스트가 있을 때,
   When 대시보드/기록/설정 전환,
   Then “Google 연결됨” 관련 라벨은 로고 하단 텍스트(`sessionStatusText`) 영역에서만 노출되고 상단 카드별 중복 표시는 사라져야 한다.

3. Given 화면 너비가 모바일일 때,
   When 탭으로 이동이 필요할 때,
   Then 사이드바 메뉴(햄버거/닫기 포함)로 이동이 가능해야 한다.

## 실패 테스트
- 모바일 탭이 남아 있으면 실패.
- 상단에 상태 텍스트가 다중 노출되면 실패.
- 햄버거 없이 모바일에서 라우트 이동이 불가능하면 실패.

## 완료 기준
- `apps/web/src/components/app-nav.tsx`와 `apps/mobile/App.tsx`에 사이드바/메뉴 동작 반영.
- `apps/web/app/globals.css`에서 모바일 네비게이션 오버레이/사이드바 클래스 반영.
- `RoutineMate 로고 아래 상태표시`는 상단 brand block에서만 존재.
