# S5-2 Spec - 웹/모바일 Google 인증 세션 유지

## 범위
- 세션 쿠키 `secure` 옵션을 환경 기반으로 분기한다.
  - 개발/로컬 HTTP: `secure=false`
  - 운영 HTTPS: `secure=true`
- `GET /v1/auth/session`은 쿠키가 없을 때 `sessionId` 쿼리 fallback을 지원한다.
- 모바일은 `expo-secure-store`에 `sessionId`를 안전 저장하고 앱 재실행 시 복구한다.

## Given / When / Then
1. Given 웹에서 Google 로그인 완료, When 페이지 새로고침, Then 재로그인 없이 세션이 복구된다.
2. Given 모바일 앱 재실행, When 쿠키가 없더라도 SecureStore에 sessionId가 있으면, Then `/v1/auth/session?sessionId=...`로 세션이 복구된다.
3. Given 저장된 sessionId 복구 실패, When `GoogleSignin.signInSilently()` 가능, Then `/v1/auth/google/session`으로 자동 복구된다.

## 실패 테스트
- 저장된 sessionId가 무효하면 null 응답 후 재로그인 경로로 폴백.
- 개발 환경 세션 쿠키에 `Secure` 플래그가 강제되면 실패(회귀 방지 테스트).

## 완료 기준
- 웹 새로고침/모바일 앱 재실행 이후 Google 세션 유지율이 향상되고 불필요한 재로그인이 제거된다.
- `/v1/auth/session?sessionId=...` 복구 및 쿠키 옵션 테스트가 green.
