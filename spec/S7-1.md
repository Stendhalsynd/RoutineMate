# S7-1 Firebase Foundation

## Goal
- Add the Firebase foundation required to replace the current Notion-backed backend for web and mobile.

## Scope
- Add Firebase and Firestore testing dependencies to the workspace.
- Add shared Firebase config helpers for web and mobile.
- Add Firestore rules and emulator-friendly test scaffolding.
- Add `.env.example` and operator docs for Firebase credentials and local development.

## Acceptance Criteria
1. Root and app package manifests declare Firebase dependencies needed for web, Expo, and rules testing.
2. The repository contains Firestore rules and Firebase project config files required for local emulation and later deployment.
3. Shared config helpers can read required Firebase env vars without leaking secrets into source.
4. A failing-then-passing test proves the foundation files and dependency declarations exist.
5. Documentation explains how to configure Firebase for RoutineMate web and mobile environments.

## Verification
- Package manifest tests pass.
- Firebase foundation tests pass.
- Type checks pass for new config files.
