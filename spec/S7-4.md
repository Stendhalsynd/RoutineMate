# S7-4 Firestore Data Repository Cutover

## Goal
- Replace the slow Notion-backed record/settings data access path with a Firestore-backed repository while keeping the current session-based API contract intact.

## Scope
- Add a Firestore data repository for meal logs, meal checkins, workout logs, body metrics, goals, templates, and reminder settings.
- Compose the runtime `repo` so session/auth methods stay legacy-compatible while data CRUD can switch to Firestore via environment configuration.
- Preserve current API response shapes so web/mobile clients do not need a breaking contract change for this slice.

## Acceptance Criteria
1. `repo` can delegate record/settings CRUD and list operations to Firestore when Firebase server config is available.
2. Existing auth/session route behavior remains unchanged in test mode.
3. Dashboard/day/reminder routes continue to pass against the current test suite.
4. The cutover can be enabled with `ROUTINEMATE_REPO_MODE=firebase` (or equivalent Firebase admin env presence).

## Verification
- `npm run test --workspaces --if-present`
- `npm run typecheck`
- `npm run build --workspace @routinemate/web`
