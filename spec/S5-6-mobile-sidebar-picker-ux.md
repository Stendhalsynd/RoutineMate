# S5-6 Mobile Sidebar + Picker UX 개선

## 배경
모바일 상단 헤더의 Google 버튼 배치, 사이드바 조작성, 체중/체지방 입력 방식, D-day 선택 UX에서 사용성 이슈가 반복됨.

## 범위
- 모바일 상단 헤더의 Google 버튼 제거, 사이드바 액션으로 이동.
- 사이드바를 드래그 제스처로 열고 닫을 수 있게 개선.
- 체중/체지방 입력을 정수/소수 분리 선택 방식으로 변경.
- D-day 입력을 스크롤 리스트에서 달력(DateTimePicker) 선택 방식으로 변경.
- 모바일 `RoutineMate` 로고 클릭 시 대시보드로 이동.
- 대시보드 하단 목표 요약에 주간 목표 + 목표 체중 + 목표 체지방 표시.
- 모바일 네트워크 실패 메시지(`Network request failed`) 사용자 친화 문구로 매핑.

## Given / When / Then
1. Given 모바일에서 기록/설정 화면일 때,
   When 체중/체지방 값을 입력하면,
   Then 정수와 소수(0~9)를 각각 스크롤 선택해 `0.1` 단위 값을 만들 수 있어야 한다.

2. Given 모바일에서 목표 D-day를 설정할 때,
   When 날짜 선택을 누르면,
   Then 일자 스크롤 대신 달력 기반 선택기가 열려야 한다.

3. Given 모바일에서 사이드바를 사용할 때,
   When 화면 가장자리/사이드바를 수평 드래그하면,
   Then 패널이 부드럽게 열리고 닫혀야 한다.

4. Given 앱 재실행 직후 네트워크가 일시 불안정할 때,
   When 초기 동기화 요청이 실패하면,
   Then 원문 에러 대신 사용자 친화 메시지를 노출해야 한다.

5. Given 모바일 상단 로고를 누르면,
   When 현재 탭이 기록/설정이라도,
   Then 대시보드 탭으로 전환되어야 한다.

## 실패 테스트
- 헤더에 Google 버튼이 남아 있으면 실패.
- 체중/체지방 입력이 단일 장문 리스트만 제공되면 실패.
- D-day가 달력 선택이 아닌 스크롤 리스트로만 동작하면 실패.
- `Network request failed` 원문이 사용자에게 그대로 노출되면 실패.

## 완료 기준
- `apps/mobile/App.tsx`에서 사이드바/로고/입력/D-day/메시지 UX가 반영됨.
- 숫자 분리 선택 유틸 및 네트워크 에러 매핑 유틸 단위 테스트가 통과함.
- 웹 모바일 사이드바 드래그 유틸 테스트가 통과함.
